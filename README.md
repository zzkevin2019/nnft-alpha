# nnft-alpha

Simulation code and data for *Optimal Architecture and Fundamental Bounds
in Neural Network Field Theory* by Zhengkang Zhang ([arXiv:2604.27050](https://arxiv.org/abs/2604.27050)).

Neural network field theory (NNFT) represents a scalar field as a neural
network and samples field configurations by drawing network parameters.
This repository implements the Cos-Net architecture [[Halverson 2112.04527](https://arxiv.org/abs/2112.04527);
[Demirtas et al. 2307.03223](https://arxiv.org/abs/2307.03223)] for a massive scalar field in `d` dimensions,
parameterized by a previously unexplored exponent `alpha` that controls
how the momentum-space integrand is split between the neuron-momentum
distribution and the neuron amplitude. All choices of `alpha` yield the
same `N → ∞` theory; finite-`N` behavior is `alpha`-dependent. The code
computes 2- and 4-point correlation functions by Monte Carlo over network
ensembles, and the included data plus `letter_plot.ipynb` reproduces the
figure in the paper.

Throughout the code, all dimensionful quantities are in units of the
scalar mass `m`; e.g., distances are in units of `1/m`, and momenta--
including the UV cutoff `Λ`--are in units of `m`.

## Installation

```bash
git clone https://github.com/zzkevin2019/nnft-alpha.git
cd nnft-alpha
pip install -r requirements.txt
```

Python 3.9+ is recommended. No compilation step; the code is pure
NumPy/SciPy.

## Reproducing the figure in the paper

The figure is generated by
`letter_plot.ipynb` from the data in `data/scan_1e8/`
(27 files, `d=2`, `Λ=100`, `N ∈ {30, 50, 100}`, `α ∈ {-1, 0, 1}`,
`r ∈ {0.3, 1, 3}`, ensemble size `M=10⁸`).

## Regenerating the data

The data files are included in the repo so the figure can be reproduced
without re-running the Monte Carlo. To regenerate them from scratch into
a separate directory (so the shipped copy is not overwritten):

```bash
python run_parameter_scan.py \
    --d 2 --N 30 --M 1e8 \
    --alphas -1 0 1 --Lambdas 100 \
    --rescales 0.3 1.0 3.0 \
    --n_configs 50 \
    --output_dir data/scan_1e8_regen

# Repeat with --N 50 and --N 100.
```

To plot from the regenerated set, either replace the contents of
`data/scan_1e8/` with `data/scan_1e8_regen/` or edit `data_dir` in
`letter_plot.ipynb` to point at the new directory.

Each `N` value runs three parameter-set workers in parallel (one per
`α`). Per-worker wall-clock time at `M=10⁸` is several hours on a modern
multi-core workstation.

Outputs are written as `G4_d{d}_N{N}_L{Lambda}_a{alpha}_r{rescale}_M1e8.npz`.
Each `.npz` contains:

- `configs` — the `(n_configs, 4, d)` coordinate tensor, already rescaled;
- `means`, `errors` — MC estimate and standard error of the 4-point function per configuration;
- `theory_reg` — free-theory 4-point function evaluated with the regularized propagator;
- `theory_inf` — free-theory 4-point function in the infinite-`Λ` limit;
- scalar metadata: `d`, `M`, `N`, `Lambda`, `alpha`, `rescale`, `seed`, `regulator`.

## Code layout

```
nnft/
├── theory.py        # Omega integrals, propagators, Wick-theorem 4-pt function
├── sampling.py      # inverse-CDF builder and NetworkGenerator
├── correlators.py   # MC estimators + parallel parameter-set driver
├── configs.py       # random four-point configuration generator
└── io.py            # filename/path helpers
run_parameter_scan.py               # CLI wrapper around compute_four_point_parameter_scan
letter_plot.ipynb                   # generates the figure in the paper
data/scan_1e8/                      # the 27 .npz files read by the notebook
n_extrapolation_d2.pdf              # figure in the paper showing NNFT four-point function for the first 5 configs
n_extrapolation_d2_all_configs.pdf  # figures showing results for all 50 configs
```

The public API is re-exported at package level, so e.g., `from nnft import
NetworkGenerator, generate_four_point_configs` resolves directly.
